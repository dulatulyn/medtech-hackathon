# Backend conventions

Layered FastAPI backend. Keep the data flow strict — no layer skipping.

```
Controller (schemas)  →  Service (DTOs)  →  Repository (models)
   src/api/v1/             src/services/        src/repositories/
```

## Layers

- **Controllers** (`src/api/v1/`): HTTP only. Validate input with Pydantic **schemas**
  (`src/api/v1/schemas/`), call a service, return its result. No business logic, no SQL.
- **Services** (`src/services/`): business logic + authorization. Accept and return **DTOs**
  (`src/dtos/`) only — never schemas, never ORM models.
- **Repositories** (`src/repositories/`): all DB access. Work with SQLAlchemy **models**
  (`src/models/`). Return models; do not raise HTTP errors.

## Data object conversion

- Controller converts schema → DTO with `SomeDTO(**schema.model_dump())`.
- Service converts model → DTO explicitly (dataclass DTOs in `src/dtos/`).
- Never return an ORM model from a service or controller.

## Dependency injection (Dishka)

- All wiring lives in `src/ioc/`. Providers: `DatabaseProvider` (engine + sessionmaker at
  APP scope, `AsyncSession` at REQUEST scope), `RepositoryProvider`, `ServiceProvider`.
- Routers use `route_class=DishkaRoute`; inject with `FromDishka[T]`.
- Add a new service/repository by registering it in the matching provider — nothing else.

## Models

- Inherit `Base`; use `ULIDMixin` (26-char string PK) and `TimestampMixin` (`created_at`,
  `updated_at`) from `src/models/base.py`.
- Register every new model's import in `alembic/env.py` so autogenerate sees it.

## Errors

- Raise `ValueError` (or a domain exception) in services; the handlers in
  `src/api/exceptions/exception_handlers.py` map them to sanitized HTTP responses.
- Responses are auto-wrapped as `{"data": ...}` by `StandardResponseMiddleware`.

## Style

- Type hints everywhere. Docstrings short (1 line). No commented-out code.
- Structured logs only: `logger.info("event_name", key=value)` — no f-strings as the event.

## Migrations

- `make migrate msg="..."` to autogenerate, `make migrate-upgrade` to apply.
- Review autogenerated SQL before committing.
